## Возможности

- **Анализ сайтов** - автоматическое определение типа сайта
- **Несколько методов парсинга:**
  - Статический HTML (requests + BeautifulSoup)
  - Динамический (Selenium + Chrome)
  - API (прямые запросы к API)
  - OCR (распознавание текста на изображениях)
  - Scrapy (промышленный парсинг)
- **Экспорт данных** - CSV, JSON, Excel, SQLite
- **Управление профилями** - сохранение и загрузка настроек
- **Планировщик задач** - автоматический парсинг по расписанию
- **Менеджер прокси** - ротация прокси-серверов
- **Поиск селекторов** - визуальный выбор элементов на странице

## Установка

### 1. Установите зависимости

```bash
pip install -r requirements.txt
```

### 2. Установите дополнительные компоненты (по желанию)

**Для OCR (распознавание текста на изображениях):**
- Скачайте Tesseract OCR: https://github.com/UB-Mannheim/tesseract/wiki
- Укажите путь к tesseract.exe в настройках программы

**Для динамического парсинга:**
- Установите Google Chrome
- selenium автоматически загрузит ChromeDriver

## Пошаговое руководство

### Шаг 1: Анализ сайта

1. Перейдите на вкладку **«🔍 Анализ»**
2. Введите URL сайта (например, `https://lenta.ru`)
3. Нажмите кнопку **«🔍 Анализировать»**
4. Дождитесь результата анализа

Программа определит:
- ✅ Есть ли JavaScript на сайте
- ✅ Есть ли формы
- ✅ Есть ли API
- ✅ Есть ли изображения с текстом
- ✅ Есть ли карта сайта (sitemap.xml)
- ✅ Динамический ли контент

### Шаг 2: Выбор методов парсинга

После анализа программа предложит подходящие методы. Выберите нужные методы:

| Метод | Описание | Пример сайтов |
|-------|---------|-------------|
| Статический | Обычный HTML | Блоги, Википедия |
| Динамический | JavaScript | VK, Facebook |
| API | Прямое API | Яндекс.Погода |
| OCR | Текст на картинках | Сканеры, фото |
| Scrapy | Промышленный | Большие сайты |

### Шаг 3: Настройка парсинга

1. Перейдите на вкладку **«⚙️ Парсинг»**
2. Введите CSS-селекторы для нужных элементов
3. Настройте параметры:
   - **Время ожидания** - сколько ждать загрузки (для динамического парсинга)
   - **Headless** - запускать Chrome без окна (для динамического)
4. Нажмите **«🚀 Запустить парсинг»**

### Шаг 4: Просмотр результатов

1. Результаты появятся на вкладке **«📊 Результаты»**
2. Данные отображаются в виде таблицы
3. Для экспорта нажмите нужную кнопку:

## Методы парсинга

### Статический парсинг

Использует `requests` + `BeautifulSoup`.

**Когда использовать:**
- Обычные HTML-страницы без JavaScript
- Простые сайты с серверным рендерингом

**Пример селектора:**
```
h1.title          # Заголовок
div.content       # Контент
a.news-link       # Ссылки
```

### Динамический парсинг

Использует `Selenium` + `Chrome`.

**Когда использовать:**
- Сайты с AJAX
- React, Vue, Angular
- Бесконечная прокрутка
- Авторизация

**Параметры:**
- Время ожидания (сек): 5-30
- Headless: да (без окна) / нет (с окном)

### API парсинг

Прямые запросы к API сайта.

**Когда использовать:**
- Сайт предоставляет API
- Нужны большие объёмы данных

**Пример:** `https://api.example.com/data`

### OCR парсинг

Распознаёт текст на изображениях через Tesseract.

**Когда использовать:**
- Текст на картинках
- Сканованные документы
- Графики с подписями

**Требования:** Установленный Tesseract OCR

### Scrapy парсинг

Промышленный фреймворк для парсинга.

**Когда использовать:**
- Большие сайты (1000+ страниц)
- Множество страниц
- Сложная логика пагинации

**Формат селекторов:**
```
title: h1.title, price: .price, description: .desc
```

## Экспорт данных

На вкладке **«📊 Результаты»** нажмите нужную кнопку:

| Формат | Кнопка | Описание |
|-------|--------|----------|
| CSV | 💾 Сохранить как CSV | Excel-совместимый |
| JSON | 💾 Сохранить как JSON | Структурированный |
| Excel | 💾 Сохранить как Excel | .xlsx файл |

### Формат файлов

Имена файлов создаются автоматически:
```
lenta.ru_20260418_231425.csv
```

Где:
- `lenta.ru` - домен сайта
- `20260418` - дата (YYYYMMDD)
- `231425` - время (HHMMSS)

## Профили и планировщик

### Профили

Сохраняйте настройки для повторного использования:

1. Настройте парсинг
2. Перейдите на вкладку **«📋 Профили»**
3. Нажмите **«💾 Сохранить текущий как профиль»**
4. Введите имя профиля

**Загрузка профиля:**
1. Выберите профиль из списка
2. Нажмите **«📂 Загрузить профиль»**
3. Нажмите «Да» для запуска анализа

### Планировщик

Автоматический запуск парсинга по расписанию:

1. Перейдите на вкладку **«⏰ Планировщик»**
2. Нажмите **«➕ Добавить задачу»**
3. Настройте:
   - **Название**: «Новости cadaver каждый час»
   - **URL**: https://example.com/news
   - **Методы**: static, api
   - **Расписание**:
     - `interval 60` - каждые 60 минут
     - `daily 14:30` - каждый день в 14:30
     - `weekly mon 09:00` - каждый понедельник в 9:00
   - **Формат экспорта**: csv, json, excel, sqlite

Вкладка **«⚙️ Настройки»**:

- Движок: Tesseract
- Языки: `rus+eng` (через запятую)

## Устранение проблем

### «Превышено время ожидания»

**Причина:** Сайт ��едленно отвечает или заблокирован.

**Решение:**
1. Проверьте URL
2. Увеличьте таймаут в настройках
3. Используйте прокси
4. Попробуйте позже

### «Tesseract not found»

**Причина:** Не установлен Tesseract OCR.

**Решение:**
1. Скачайте: https://github.com/UB-Mannheim/tesseract/wiki
2. Установите
3. Укажите путь в настройках

### Динамический парсинг не работает

**Причина:** Не установлен Chrome или ChromeDriver.

**Решение:**
1. Установите Google Chrome
2. Попробуйте отключить headless-режим для отладки
3. Проверьте, что ChromeDriver совместим с вашей версией Chrome

### Парсинг не находит данные

**Причина:** Неправильный селектор.

**Решение:**
1. Откройте сайт в браузере
2. Нажмите F12 для инструментов разработчика
3. Используйте вкладку **«Поиск селекторов»** для визуального выбора
4. Протестируйте селектор

### Ошибка доступа (403, 429)

**Причина:** Сайт блокирует запросы.

**Решение:**
1. Добавьте задержку между запросами
2. Используйте прокси
3. Измените User-Agent
4. Включите ротацию прокси
